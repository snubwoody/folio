# Contributing to Folio

Folio is an open-source project, and we welcome contributions from anyone who is interested in helping improve the app. Whether you're a developer, a translator, or a documentation writer, there are many ways to get involved.

## Contributing Guidelines

[Branching strategy?]

Before you submit a pull request to Folio, please ensure that you have followed these guidelines:

- Code should be well-documented and formatted.
- All changes should be covered by tests.
- Commits should be well-written and descriptive, with a clear summary of the changes made and any relevant context.
- Pull requests should target the `main` branch and include a clear summary of the changes made.

## Bug Reports and Feature Requests

When requesting new features, please open a GitHub Discussion under the [Ideas](https://github.com/snubwoody/folio/discussions/categories/ideas) category first. This allows us to work on the design before you start writing code, and prevents you from possibly wasting your time.

When submitting a bug report, include your platform and the version of Folio the bug occurred on. Bug reports must include minimal steps to reproduce.
